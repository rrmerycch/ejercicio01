# LEYES UX/UI PARA PRINCIPIANTES

## ¿Qué son las leyes UX/UI?

Son principios de diseño que ayudan a crear páginas y aplicaciones fáciles de usar, entender y navegar.

Se utilizan para mejorar la experiencia del usuario y evitar confusión, errores o frustración.

---

# 1. LEY DE HICK

## Concepto

La Ley de Hick dice que mientras más opciones tenga una persona, más tiempo tardará en decidir.

Cuando una pantalla tiene demasiados botones, enlaces o decisiones, el usuario puede sentirse confundido o abrumado.

Por eso las interfaces deben mostrar solo las opciones importantes.

## Ejemplo

❌ Malo:

```txt
20 botones diferentes
```

✅ Bueno:

```txt
Inicio | Productos | Contacto
```

---

# 2. LEY DE FITTS

## Concepto

La Ley de Fitts explica que los elementos grandes y cercanos son más fáciles y rápidos de seleccionar.

Esto es especialmente importante en celulares, donde tocar botones pequeños puede ser incómodo.

Los botones importantes deben ser visibles y fáciles de alcanzar.

## Ejemplo

❌ Malo:

```txt
[x]
```

✅ Bueno:

```txt
[ COMPRAR ]
```

---

# 3. LEY DE JAKOB

## Concepto

La Ley de Jakob dice que las personas esperan que las páginas funcionen parecido a otras páginas conocidas.

Los usuarios ya están acostumbrados a ciertas estructuras como:

- menú arriba
- logo arriba izquierda
- búsqueda visible

Cambiar eso innecesariamente puede generar confusión.

## Ejemplo

✅ Bueno:

```txt
Logo arriba
Menú visible
```

❌ Malo:

```txt
Menú escondido sin razón
```

---

# 4. LEY DE PROXIMIDAD

## Concepto

La Ley de Proximidad dice que los elementos cercanos visualmente parecen relacionados.

Agrupar información ayuda a que las personas entiendan mejor el contenido y encuentren información más rápido.

## Ejemplo

✅ Bueno:

```txt
Nombre
Correo
Teléfono
```

❌ Malo:

```txt
Nombre


Correo



Teléfono
```

---

# 5. LEY DE SIMILITUD

## Concepto

La Ley de Similitud dice que los elementos que se ven parecidos parecen pertenecer al mismo grupo.

Usar los mismos colores, tamaños y estilos ayuda a mantener consistencia visual y facilita comprensión.

## Ejemplo

✅ Bueno:

```txt
Botones verdes = aceptar
Botones rojos = cancelar
```

---

# 6. LEY DE MILLER

## Concepto

La Ley de Miller explica que las personas no pueden recordar demasiada información al mismo tiempo.

Por eso el contenido debe dividirse en partes pequeñas y organizadas para facilitar lectura y memoria.

## Ejemplo

❌ Malo:

```txt
928374618273
```

✅ Bueno:

```txt
928 - 374 - 618
```

---

# 7. LEY DE TESLER

## Concepto

La Ley de Tesler dice que todo sistema tiene cierta complejidad.

El objetivo del diseño es que esa dificultad no recaiga en el usuario.

Las interfaces deben simplificar procesos y reducir esfuerzo.

## Ejemplo

❌ Malo:

```txt
Formulario muy largo
```

✅ Bueno:

```txt
Ingresar con Google
```

---

# 8. LEY DE DOHERTY

## Concepto

La Ley de Doherty explica que las personas prefieren interfaces rápidas.

Cuando una página responde rápido, el usuario siente mayor comodidad y control.

Las páginas lentas generan frustración y abandono.

## Ejemplo

❌ Malo:

```txt
Carga en 10 segundos
```

✅ Bueno:

```txt
Carga inmediata
```

---

# 9. EFECTO ESTÉTICO-USABILIDAD

## Concepto

Este principio dice que las personas perciben diseños bonitos como más fáciles de usar.

Aunque dos aplicaciones funcionen igual, normalmente la más limpia y ordenada genera más confianza.

## Ejemplo

✅ Bueno:

```txt
Diseño limpio
```

❌ Malo:

```txt
Diseño desordenado
```

---

# 10. LEY DE PRÄGNANZ

## Concepto

La Ley de Prägnanz dice que las personas prefieren diseños simples y organizados.

El cerebro entiende más rápido interfaces limpias que pantallas llenas de elementos innecesarios.

## Ejemplo

❌ Malo:

```txt
Muchos colores y botones
```

✅ Bueno:

```txt
Diseño simple
```

---

# RESUMEN

| Ley | Idea principal |
|---|---|
| Hick | menos opciones |
| Fitts | botones grandes |
| Jakob | estructuras conocidas |
| Proximidad | agrupar elementos |
| Similitud | elementos parecidos |
| Miller | simplificar información |
| Tesler | reducir dificultad |
| Doherty | rapidez |
| Estético | diseño atractivo |
| Prägnanz | simplicidad |

---

# CONCLUSIÓN

Las leyes UX/UI ayudan a crear interfaces:

- fáciles
- rápidas
- claras
- cómodas
- organizadas

El objetivo principal es mejorar la experiencia de las personas al usar una página o aplicación.
